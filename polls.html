Manager Ballot
Each manager should get a unique ballot link or code. That token is the validation layer. It is cleaner than asking people to type their name and easier to lock to one response.
Validation Model
  • One generated token per manager per poll.
  • Backend resolves token to a single manager identity.
  • Unique database constraint blocks duplicate manager ballots.
  • Optional resubmission means edits overwrite prior answers until the deadline.
Live Results
Native static page, local question file, external vote storage. The page stays simple because the backend only has to do three things: resolve a ballot token, accept a ballot, and return aggregates.
